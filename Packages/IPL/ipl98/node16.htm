Compiling the library

The library has been compiled with several compilers and environments. Here a list of links and information for the different compilers:

  • MS Visual C++ 6.0  Information about creating a console-project, window-project and GUI-related methods can be found at: ``http://www.mip.sdu.dk/ edr/visualc/''4. It should work from Visual C++ 4.2 and upwards.
  • Borland C++ Builder 4.0  A complete project can be downloaded from the official IPL98 homepage by clicking at the ``IPL98 Exchange'' button and following the link ``Borland C++ Builder GUI and IPL98''. Note that the library cannot be used without the namespace ipl with the Borland environment, this means you have to put the prefix ``ipl::'' in front of all IPL98 types and definitions.
  • GNU G++   The g++ compiler has been tested on a UNIX  machine with IPL98 v. 1.01 (it should work on a Linux  OS too). However, newer versions of IPL98 have caused problems, at the moment g++ v. 2.95.2 cannot compile the library since the std library header file os seems to be non-existent. The newer but still at the time of writing this unstable version 2.97.x should work but has not been tested. It has also been tried with Sun's CC compiler, but it seems to have problems compilating the CPoint2D and CPoint3D template based classes, it can be fixed by rearranging some code in the header files, but then it does not compile under MS Visual C++ at the moment. As soon as new versions of compilers are available we will look at this problem. Here is an example of compiling a simple example5 piece of code with g++ and IPL98 v. 1.01: 
    g++ ipl98/ipl98_cplusplus.cpp ipl.cpp -lm -Wno-non-template-friend
    This example compiles the file ipl.cpp containing some code using IPL98. The skeleton of the code in the file looks like this: 
    #include "ipl98/ipl98_cplusplus.h"
#include <stdio.h>
using namespace ipl;

void main()
{
   CImage Img,ImgDest; // Images
   ...
}
    The option ``-Wno-non-template-friend'' given to the g++ compiler is to avoid a problem with the CPoint template classes. This may be solved in a future version. Remember that because most UNIX machines are BIG ENDIAN types you have to comment in the corresponding line in the ipl98.h file as explained in chapter .
