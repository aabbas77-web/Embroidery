File names and placement

The files belonging to IPL98 is split into three folders (one of them contains several subfolders): ipl98, points and time, the hierarchy of the folders can be seen in figure . The camera-folder used to be an integrated part of IPL98 but has since version 1.10 been removed. It is still higly supported but is a seperate add on to the library, it is available for download at the IPL98 hompeag in the Camera Classes section. The reason for moving it is that its a hardware dependent area, new folders with specialized code will be added here as different framegrabbers and cameras are used, read more about the camera classes in section . The folder ipl98 contains both the C and C++ code for the containers and image classes/functions. The points folder is C structures and C++ classes giving two and three dimensional points and corresponding functions to work with the points. They can be used independently of the IPL98 library. The time folder contains C functions and a C++ class for using a timer and can also be used without using the rest of the library3. Every ANSI C code is placed in folders named kernel_c in the relevant folders.

The ipl98 folder contains most of the code in the library. The four files needed for working with the library is placed in the folder together with two folders: cpp_classes and kernel_c. The cpp_classes contains all the C++ code. Here is all container classes and a palette class. In the folder algorithms is all the classes containing code for image proccessing algorithms. The last folder named general_functions contains a file with macros used for general purposes. The file names containing classes should always have the same name as the class name except the leading letter ``C''. In the kernel_c folder are the ANSI C code. Here the different matrix image classes is grouped into folders with relevant names. The algorithms is again contained in a folder named algorithms. The file names should give an idea about the contents in them.
